# Default Library

This is the built-in unit-definition catalog shipped with the app.

The files are organized by:
1. domain folder: `air/`, `land/`, `sea/`, `subsurface/`
2. country-of-origin file: for example `usa_air_units.yaml`

Definitions here should represent operational entities at the strategic level. Avoid splitting one real-world battery, ship, or base into low-level support subcomponents unless the gameplay explicitly needs it.
